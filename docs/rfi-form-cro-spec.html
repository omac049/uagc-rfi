Key Takeaways

  • Support bubbles are a 4-component contextual help system across a 2-phase form flow
  • Phase 1: Program Selection (education level + degree) → Phase 2: Contact Info + Military Status + Start Timeline → Thank You Page
  • Goal: reduce form abandonment by 5-15%
  • Primary metric: form completion rate
  • 3-variation A/B test via Optimizely

What Are Support Bubbles?

Support bubbles are a contextual help system designed to guide users through form completion, reduce form abandonment, provide just-in-time assistance, and encourage users who are struggling.

Component Architecture

How the support bubble system is structured
ComponentLocationRelationship
Smart Tip ContainerInside the RFI form, above the form fieldsDisplays contextual messages (info / success / warning) inline
Form Fields with TooltipsBeside each field labelSmall (?) icons that expand on hover/click to show field-specific help
Floating Help ButtonFixed, bottom-right corner of the viewportAlways visible; opens the Help Modal on click
Help ModalFull-screen overlay (centered)Step-specific comprehensive help content; triggered by the floating button

⚠️ Engagement Gate

Help messages ONLY appear AFTER the user has engaged with the form (first focus or click on any form field). This prevents intrusive pop-ups on page load and ensures a non-disruptive experience.

QA Checklist

Quality assurance verification items

QA Checklist

  • Smart tip container appears on all defined triggers
  • Smart tip auto-dismisses after 10 seconds (info/success types)
  • Warning tips persist until manually closed by user
  • Floating help button visible and clickable on all pages
  • Help badge appears after 60 seconds of inactivity
  • Help modal opens/closes correctly (overlay, × button, Escape)
  • Help modal content switches based on current phase (Phase 1 vs Phase 2)
  • Tooltips work on hover (desktop) and click (mobile)
  • All events fire correctly to GA4
  • Mobile responsive design works (test 320px-768px)
  • Accessibility: all focus states, ARIA labels present
  • No JavaScript errors in browser console
  • Performance: no visible lag, jank, or layout shift
  • Print stylesheet hides all support bubble elements
  • Form still submits correctly with support bubbles active
  • All 41 “Keep” fields present in Lead API payload on submission
  • Hidden fields (uniqueLeadId, clientid, device_type, etc.) auto-populated correctly
  • Experiment ID / Experience Variation fields populated from Optimizely
  • Partner ID & Partner Name populated on partnership landing pages
  • GA analytics fields (ga_source, ga_medium, ga_timestamp) captured accurately
  • 12 deprecated fields confirmed removed from payload
